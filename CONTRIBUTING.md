# Contributing to BONSAI

BONSAI is currently developed in a private repository. An invitation to review or contribute does not authorize public disclosure, external upload, or execution beyond an approved STS scope.

## License

Unless explicitly marked otherwise, any contribution intentionally submitted for inclusion in BONSAI is licensed under either of:

- Apache License, Version 2.0 (`Apache-2.0`); or
- MIT License (`MIT`).

at the recipient's option. The repository SPDX expression is `MIT OR Apache-2.0`. By contributing, you represent that you have the right to submit the work under those terms.

## Change discipline

1. Read the [source-of-truth governance](docs/governance/SOURCE-OF-TRUTH.md) and approved PSPR.
2. Work only within an explicitly authorized prompt and an isolated worktree.
3. Preserve the OaK/BONSAI attribution boundary and Track A no-replay semantics.
4. Record reuse, provenance, dependencies, and relevant licenses.
5. Run the prompt gate and applicable universal checks; do not hide failed seeds, unavailable measures, or ignored tests.
6. Keep secrets, personal paths, hostnames, serial numbers, raw environment dumps, and credentials out of commits and evidence.
7. Use one focused commit per prompt unless the DEVLOG justifies an inseparable bundle.

Acceptance into a private branch is not approval to publish or claim instrument/capability completion.
